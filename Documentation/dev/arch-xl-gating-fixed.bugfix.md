Per-arch compile flags are now target-scoped on `ROSS` rather than injected into the global `CMAKE_C_FLAGS`. The IBM XL optimization flags (`-O5 -qprefetch=aggressive -qarch=pwr9 ...`) are gated on **both** `CMAKE_C_COMPILER_ID MATCHES "^XL"` and `CMAKE_SYSTEM_PROCESSOR STREQUAL "ppc64le"`, so ppc64le-with-gcc builds (the typical configuration on current Power9/Power10 systems) no longer error on flags gcc doesn't understand.
